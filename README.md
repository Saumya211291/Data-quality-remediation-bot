# Data Quality Remediation Bot

This small Streamlit app accelerates root-cause analysis (RCA) for failed records generated by data-quality engines such as Dataplex, Atacama, or similar.

## Problem statement

During typical ETL workflows, data engineers use various DQ engines to validate incoming data. The engine outputs a CSV listing all records that violated one or more rules. At that point the engineer must:

1. Figure out *why* each record failed (pipeline bug? source data issue?)
2. Locate the underlying table and column
3. Determine the proper fix for the record (correct value, set `NULL`, request source-team update, etc.)
4. Reload the corrected records

This manual, collaborative process often takes a lot of time and back-and-forth with the source file teams.

## How the bot helps

1. **Upload CSV of failed records** – the only input required is the DQ engine output.
2. **Automated RCA** – the bot sends the records to a Gemini model with instructions to inspect each row, identify the problematic table/column, classify the DQ dimension, and suggest a fix.
3. **Readable report** – a detailed paragraph is generated per record, explaining what went wrong and how to correct it.
4. **Interactive follow-ups** – after the initial report the user can ask additional questions about the uploaded records (e.g. "why did record 5 fail?") and the bot will answer using the same CSV context.

The tool dramatically reduces the time required to diagnose and remediate failed records, allowing data engineers to focus on correcting the pipeline instead of manual analysis.

## Usage

1. Clone the repository and create a virtual environment:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate.ps1    # Windows PowerShell
   pip install -r requirements.txt
   ```
2. Add a `.env` file in the project root with:
   ```text
   GOOGLE_API_KEY=your_api_key_here
   GENAI_MODEL=gemini-2.5-flash
   ```
3. Run the app:
   ```bash
   .venv\Scripts\python -m streamlit run app.py
   ```
4. In your browser, upload the CSV containing failed records.
5. Wait for the remediation report, then ask follow-up questions if needed.

## Architecture

- `app.py` defines the Streamlit UI:
  - Upload widget
  - Report display
  - Optional chat input for questions
- `utils.py` handles communication with the Gemini model, context management, and CSV analysis logic.

## Future upgrades

- **Automated correction file**: generate a new CSV containing corrected records based on the model's suggestions and optionally reload it into the target table.
- **Model selection & prompt tuning**: allow users to choose different models or adjust the prompt for specific domains.
- **Structured output**: return JSON or YAML for easier downstream processing.

## Notes

- Currently uses the deprecated `google.generativeai` package; migrating to `google.genai` is recommended when it stabilizes.
- The app strips quotes from API keys and loads `.env` variables automatically.

---

This project is meant to be a lightweight, extendable starting point for accelerating data-quality remediation workflows using generative AI.
